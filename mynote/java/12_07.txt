11)Inheritance
:타클래스에서 데이터와 메소드를 상속받아 사용, 변경이 가능하다.

-접근제어자 class 이름 extends 상위클래스(){}
>재사용으로 새로운 클래스 작성, (조상, 자손)관계
>타클래스 활용으로 높은 수준의 결과물 작성 가능

-추상화:다양한 객체를 하나의 객체로 묶을 수 있다.

-Object 클래스
:조상이 없는 클래스는 자동으로 11개의 메소드를 상속받는다.

-오버로딩: 같은 이름의 메소드를 중복으로 정의
-오버라이딩: 부모 클래스의 메소드를 자식 클래스에서 재정의. 상속 후에 자식 클래스가 없는 메소드를 호출하면 부모 클래스에서 호출한다. 재정의시 자식 클래스의 메소드를 사용한다.

생성자 문제
:생성자가 존재하면 JVM이 자동으로 생성해주지 않는다. 수퍼 클래스에 매개변수가 있는 생성자를 정의하면 자동으로 매개변수가 있는 생성자로 호출한다.

-super 메소드이름(매개변수)
:자식 클래스에서 부모 클래스를 사용하고자 할때 사용한다.
>super(매개변수) > 반드시 첫번째 라인에 와야한다.
